# UM982 GNSS Receiver

How to configure the Unicore UM982 on the ArduSimple simpleRTK3B Compass for
BlueBoat use: dual-antenna heading, 10 Hz position/velocity, standalone
operation, with hooks for future NTRIP corrections.

---

## 1. Overview

This guide produces a UM982 configured to stream BESTNAVA, UNIHEADINGA, and
BESTVELA at 10 Hz plus 1 Hz diagnostics, saved to flash, ready for the boat's
nav stack to consume. Configuration is done with the uPrecise serial terminal
(§4–§5). Antenna baseline and heading-offset calibration live in
[`17_calibration.md`](17_calibration.md) and are referenced here.

## 2. Prerequisites

- UM982 on the simpleRTK3B Compass installed and wired
  ([`11_electrical_wiring.md`](11_electrical_wiring.md) §8).
- Antenna baseline measured (see [`17_calibration.md`](17_calibration.md)).
- A laptop running **uPrecise** (Unicore's configuration tool) with the module
  reachable on its USB serial COM port at **230400 baud, 8N1**.
- Firmware target: R4.10 Build 11826 or later.

## 3. Context

### 3.1 Why Proprietary Messages over NMEA

The nav stack consumes Unicore's binary/ASCII messages (BESTNAVA, UNIHEADINGA,
BESTVELA) rather than generic NMEA (GGA/RMC) because they carry the fields the
stack needs natively: per-component position and velocity uncertainties,
explicit solution-status and position-type enums, and dual-antenna heading with
its own standard deviation — none of which NMEA exposes cleanly.

### 3.2 Mode Selection

The receiver runs `MODE ROVER UAV`. Boat dynamics fall well within the UAV
motion envelope, and UAV mode does not over-constrain the solution the way
SURVEY (static) or AUTOMOTIVE (ground-vehicle assumptions) would.

### 3.3 Heading Geometry

Heading is derived from the two antennas: the rear antenna is master, the front
is slave, and the heading vector points master → slave. The baseline length and
heading offset come from calibration ([`17_calibration.md`](17_calibration.md))
and are entered with `CONFIG HEADING LENGTH`.

### 3.4 Standalone Optimization

For standalone (no-RTK) operation on water the config uses dual-frequency
multi-constellation PVT (`CONFIG PVTALG MULTI`), a 10° elevation mask, and a
35 dB-Hz C/N0 mask. The elevation and C/N0 masks drop low-elevation and weak
satellites that contribute more multipath and atmospheric error than useful
geometry — both common near a low, reflective water horizon.

## 4. Connect to the Receiver

Configuration is performed through the **uPrecise** serial terminal — the
validated method for the Greece fleet.

1. Connect the module to the laptop over USB; note the COM port it enumerates
   as.
2. In uPrecise, connect to that COM port using **auto-baud** detection. The
   modules ship at a **slower default baud** (confirm the shipped value — see
   §11), so connecting at a fixed 230400 will fail until the receiver's baud is
   changed.
3. Open uPrecise's command/serial terminal and send `VERSION`; confirm the
   firmware is R4.10 Build 11826 or later.
4. Raise the receiver's COM1 baud to 230400, then reconnect at the new rate:

   ```text
   CONFIG COM1 230400
   ```

   After the `OK*xx`, reconnect uPrecise to the same COM port at **230400,
   8N1**. (USB CDC ignores the host-side baud, but the receiver stores 230400;
   it matters if the UM982 is ever driven over a real UART.) The §5 commands are
   sent at 230400.

> **Note.** A headless, scriptable direct-serial method (sending the same
> commands over `/dev/ttyACM0` from the Pi, no GUI) would be useful for fleet
> automation but has **not been validated** — see §11.

## 5. Apply the Configuration

In the uPrecise terminal, send the commands one at a time, waiting for the
`OK*xx` acknowledgment between sends. Each command is terminated with CRLF
(`\r\n`).

```text
MODE ROVER UAV

CONFIG HEADING FIXLENGTH
CONFIG HEADING LENGTH <baseline_cm> 1
CONFIG HEADING RELIABILITY 3

CONFIG PVTALG MULTI
MASK 10
MASK CN0 35
CONFIG SMOOTH HEADING 1
CONFIG SIGNALGROUP 4 5
CONFIG SBAS DISABLE

BESTNAVA 0.1
UNIHEADINGA 0.1
BESTVELA 0.1

STADOPA 1
STADOPHA 1
RTKSTATUSA 1
BESTSATA 1

SAVECONFIG
```

> **Critical.** Replace `<baseline_cm>` with the actual antenna baseline
> measured in [`17_calibration.md`](17_calibration.md). The per-fleet default
> is TBD; capture it in
> [`01_fleet_and_network_reference.md`](01_fleet_and_network_reference.md) once
> measured.

## 6. Verification

1. Outside with a clear sky view, confirm BESTNAVA emits at 10 Hz.
2. Solution status reaches `SOL_COMPUTED`; position type `SINGLE` at minimum.
3. UNIHEADINGA shows `sol_stat == SOL_COMPUTED` with `hdgstddev` < 1°.
4. The `length` field in UNIHEADINGA matches the measured baseline within
   tolerance.
5. Velocity (BESTVELA `hor_spd`) tracks boat motion when the boat is moved.

## 7. Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| uPrecise won't connect / no response | Baud mismatch (module at slower default) | Connect with auto-baud, then `CONFIG COM1 230400` and reconnect at 230400 (§4). |
| `length` drifts from baseline | Loose antenna or wrong `CONFIG HEADING LENGTH` | Re-tighten antennas; re-measure; re-issue the command. |
| No heading solution | `INSUFFICIENT_OBS` on the slave | Improve sky view on the slave antenna; check the coax. |
| Heading noisy at 10 Hz | Multipath / low-C/N0 sats | Raise `MASK CN0` to 38 or `MASK` to 12. |
| Solution flips SINGLE → COLD_START | Power glitch | Check the buck converter and USB power. |

## 8. Future NTRIP Readiness

The custom nav stack feeds RTCM bytes to the UM982 over the same USB serial
port — no receiver configuration change is needed to accept corrections. As
corrections converge, BESTNAVA `pos_type` should walk SINGLE → PSRDIFF →
L1_FLOAT → NARROW_INT, and RTKSTATUSA reports correction health. Caster
credentials live in `00_secrets.md` as `{{NTRIP_USERNAME}}` /
`{{NTRIP_PASSWORD}}`.

## 9. Reference: Key Message Fields

Summary of the fields the nav stack uses; the full field-by-field tables are in
[`14b_um982_reference.md`](14b_um982_reference.md).

- **BESTNAVA** — primary position/velocity message: solution status, position
  type, lat/lon/height with per-component standard deviations, velocity.
- **UNIHEADINGA** — dual-antenna heading: solution status, heading, pitch,
  baseline `length`, and `hdgstddev`.
- **BESTVELA** — velocity over ground, course, and vertical speed.
- **Diagnostics @ 1 Hz** — STADOPA / STADOPHA (DOP), RTKSTATUSA (correction
  health), BESTSATA (per-satellite C/N0 and use status).

## 10. Reference: Parser Notes

Unicore ASCII messages use a header, a `;` separator between header and data
fields, and a trailing CRC. Key enum values the parser keys on include
`SOL_COMPUTED` (solution status) and `SINGLE` / `PSRDIFF` / `L1_FLOAT` /
`NARROW_INT` (position type). Full framing and enum detail are in
[`14b_um982_reference.md`](14b_um982_reference.md).
