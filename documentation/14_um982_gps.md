# UM982 GNSS Receiver

How to configure the Unicore UM982 on the ArduSimple simpleRTK3B Compass
for BlueBoat use: dual-antenna heading, 10 Hz position/velocity, standalone
operation, with hooks for future NTRIP corrections.

## 1. Overview
_To do._

What this guide produces: a UM982 configured to stream BESTNAVA,
UNIHEADINGA, BESTVELA at 10 Hz plus 1 Hz diagnostics, saved to flash, ready
for the boat's nav stack to consume.

## 2. Prerequisites
_To do._

- UM982 on simpleRTK3B Compass installed and wired
  (`11_electrical_wiring.md` §8).
- Antenna baseline measured (`17_calibration.md` §_TBD_).
- Pi can reach `/dev/ttyACM0` or `/dev/ttyUSB0`.
- Terminal program with line-ending CRLF (or use the script in §6).

## 3. Context

### 3.1 Why Proprietary Messages over NMEA
_To do._

Brief on why BESTNAVA + UNIHEADINGA + BESTVELA are preferred over GGA/RMC:

### 3.2 Mode Selection
_To do._

Why `ROVER UAV` not SURVEY or AUTOMOTIVE. Boat dynamics fall well within
UAV envelope; UAV doesn't over-constrain.

### 3.3 Heading Geometry
_To do._

Rear antenna = master, front = slave. Heading vector points master →
slave. Baseline length and heading offset come from calibration.

### 3.4 Standalone Optimization
_To do._

PVTALG MULTI (dual-frequency standalone), elevation mask 10°, CN0 mask 35
dB-Hz. Why these matter on water.

## 4. Connect to the Receiver
_To do._

Step-by-step:

1. Identify the device (`ls /dev/ttyACM* /dev/ttyUSB*`).
2. Open at 230400 baud, 8N1.
3. Send `VERSION` and confirm firmware (target: R4.10 Build 11826 or later).

## 5. Apply the Configuration
_To do._

Send commands one at a time, waiting for `OK*xx` between sends:

```text
MODE ROVER UAV

CONFIG HEADING FIXLENGTH
CONFIG HEADING LENGTH <baseline_cm> 1
CONFIG HEADING RELIABILITY 3

CONFIG PVTALG MULTI
MASK 10
MASK CN0 35
CONFIG SMOOTH HEADING 1
CONFIG SIGNALGROUP 4 5
CONFIG SBAS DISABLE

BESTNAVA 0.1
UNIHEADINGA 0.1
BESTVELA 0.1

STADOPA 1
STADOPHA 1
RTKSTATUSA 1
BESTSATA 1

SAVECONFIG
```

> **Critical.** Replace `<baseline_cm>` with the actual antenna baseline
> measured in `17_calibration.md`. The current per-fleet default is TBD;
> capture it in `01_fleet_and_network_reference.md` once measured.

## 6. Verification
_To do._

1. Outside (clear sky view), confirm BESTNAVA emits at 10 Hz.
2. Solution status reaches `SOL_COMPUTED`; pos type `SINGLE` minimum.
3. UNIHEADINGA `sol_stat == SOL_COMPUTED` with `hdgstddev` < 1°.
4. `length` field in UNIHEADINGA matches the measured baseline within
   tolerance.
5. Velocity (BESTVELA `hor_spd`) tracks boat motion when moved.

## 7. Troubleshooting
_To do._

| Symptom | Likely cause | Fix |
|---|---|---|
| `length` drifts from baseline | Loose antenna or wrong CONFIG HEADING LENGTH | Re-tighten antennas; re-measure; re-issue command. |
| No heading solution | `INSUFFICIENT_OBS` on slave | Improve sky view on slave antenna; check coax. |
| Heading noisy at 10 Hz | Multipath / low CN0 sats | Raise `MASK CN0` to 38 or `MASK` to 12. |
| Solution flips SINGLE → COLD_START | Power glitch | Check buck converter and USB power. |
